error while loading shared libraries: libjli.so 问题解决
2012-11-21 | pyinx | it故障

##

报错信息：

[root@hadoop ~]# /usr/bin/java
/usr/bin/java: error while loading shared libraries: libjli.so: cannot open shared object file: No such file or directory

解决方案:



rm -rf /usr/bin/javac

rm -rf /usr/bin/jar
rm -rf /usr/bin/java

然后建立软链

ln -s /usr/local/jdk1.6.0_13/bin/javac /usr/bin/javac

ln -s /usr/local/jdk1.6.0_13/bin/jar /usr/bin/jar
ln -s /usr/local/jdk1.6.0_13/bin/javac /usr/bin/java

分析原因:


原来安装的时候,安装了openjdk,卸载的时候可能这些执行文件没卸载掉.
导致不能使用. 所以将/usr/bin/下的老执行文件删除掉就能执行了.

查找libjli.so文件

>

>
> find / -name libjli.so

>
>
ldd /usr/bin/java

>
>
mv /usr/local/jdk1.6.0_13/jre/lib/i386/jli/libjli.so /lib
